# We-Map 项目运行指南

## 项目简介

We-Map 是一个基于 **Tauri 2.x** 的桌面应用程序，前端使用 **Vue 3 + TypeScript + Vite**，后端使用 **Rust**。主要功能包括地图展示（Leaflet）和 Mapillary 街景图像浏览。

## 环境要求

| 工具 | 最低版本 | 说明 |
|------|---------|------|
| Node.js | v18+ | JavaScript 运行时 |
| npm | v9+ | 包管理器（随 Node.js 安装） |
| Rust | 1.77.2+ | Tauri 后端编译所需 |
| Cargo | 随 Rust 安装 | Rust 包管理器 |

> 本机当前版本：Node v20.9.0 / npm 10.8.2 / rustc 1.93.1 / cargo 1.93.1

## 安装依赖

在项目根目录下执行：

```bash
npm install
```

> `node_modules` 目录已存在时可跳过此步。

## 运行方式

### 1. 开发模式（桌面应用）

启动 Tauri 开发窗口，前端会自动运行 Vite dev server（端口 5173），后端编译 Rust 代码：

```bash
npx tauri dev
```

该命令会依次：
1. 执行 `npm run dev` 启动 Vite 开发服务器
2. 编译 Rust 后端代码
3. 打开桌面应用窗口

### 2. 仅前端开发（浏览器预览）

如果只想调试前端界面（不涉及 Tauri API 调用）：

```bash
npm run dev
```

然后在浏览器中访问 `http://localhost:5173`。

### 3. 构建生产版本

编译前端并打包为桌面应用程序：

```bash
npx tauri build
```

该命令会依次：
1. 执行 `npm run build`（`vue-tsc -b && vite build`）生成前端产物到 `dist/`
2. 编译 Rust 后端为 release 模式
3. 在 `src-tauri/target/release/bundle/` 下生成安装包

## 常见问题

### Rust 未安装

访问 https://rustup.rs/ 安装 Rust 工具链，安装后重启终端。

### 端口 5173 被占用

关闭占用该端口的进程，或在 `vite.config.ts` 中修改端口。

### 首次编译 Rust 耗时较长

Tauri 首次编译需要下载和编译大量 Rust 依赖，可能需要几分钟。后续编译会使用缓存，速度更快。

### Windows 缺少 WebView2

Tauri 依赖 Windows WebView2 运行时。Windows 11 已内置，Windows 10 需要手动安装：
https://developer.microsoft.com/en-us/microsoft-edge/webview2/
